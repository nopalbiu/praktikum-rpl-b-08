# Retrospektif Kelompok B-08

## What went well? 
- Pembagian tugas berjalan cukup seimbang antara pengerjaan Fullstack (Naufal), Frontend (Hesa), Backend (Yahya), dan QA (Andrew).
- Penggunaan AI sebagai asisten *coding* sangat membantu mempercepat proses pembuatan boilerplate dan styling menggunakan Tailwind CSS.
- Fitur-fitur utama (keranjang belanja, checkout, dan manajemen produk) berhasil diselesaikan tepat waktu dan terintegrasi dengan baik.

## What didn't go well? 
- Sering terjadi *merge conflict* di Git karena beberapa anggota tanpa sengaja bekerja di file yang sama tanpa komunikasi sebelumnya.
- Manajemen waktu di awal sprint kurang disiplin, sehingga pengujian (testing) dan penyusunan dokumentasi menumpuk di minggu-minggu terakhir.
- Kurangnya *unit testing* di fase awal pengembangan, sehingga beberapa *bug* (seperti input kuantitas negatif) baru ketahuan saat fase akhir integrasi.

## What can we improve?
- **Action Item 1:** Menerapkan strategi *branching* Git yang lebih disiplin (satu *branch* khusus untuk satu fitur/isu) dan melakukan *daily standup* singkat via chat grup.
- **Action Item 2:** Membiasakan budaya *Test-Driven Development* (TDD) ringan sejak awal proyek agar tim tidak kewalahan menambal *bug* di akhir.
